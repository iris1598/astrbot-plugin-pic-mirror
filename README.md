# AstrBot 图像对称处理插件

一个为 AstrBot 设计的图像对称处理插件，专门生成各种“诡异”而有趣的对称图片，希望能给大家带来一点欢乐！

> **社区力量更新**：曾经最头疼的“GIF 不动”问题，已在社区贡献者 `@superexboom` 的帮助下成功修复。感谢开源协作的力量！

## 功能特性

*   **多种图像效果**：左对称、右对称、上对称、下对称、反色，制造各种“鬼畜”效果。
*   **多图像源支持**：支持处理静态图片（PNG, JPG, WebP, BMP）、GIF 动画、QQ 用户头像。
*   **智能解析**：自动识别回复消息中的图片、同条消息中的图片以及 @ 用户头像。
*   **多适配器支持**：兼容 `aiocqhttp` 等普通 QQ 平台，并适配 `qqofficial` / `qqofficial_full` 官方机器人适配器（基于 AppID + openid 取头像）。
*   **安全防护**：内置 SSRF 防护、文件大小预检查、GIF 帧数和总像素限制。
*   **稳定性保护**：支持用户频率限制、全局并发处理上限和自动清理任务。
*   **WebUI 配置**：配置项通过 `_conf_schema.json` 暴露，可在 AstrBot Web 界面调整并由插件运行时读取。

## 快速开始

### 安装方法

1.  **从 AstrBot 插件市场安装（推荐）**
    *   在 AstrBot Web 管理界面进入“插件市场”。
    *   搜索“图像对称”或直接输入仓库地址安装。

2.  **手动安装**
    *   从 [GitHub Releases](https://github.com/FenChen0211/astrbot-plugin-pic-mirror/releases) 下载最新的 `zip` 包。
    *   在 AstrBot 插件管理界面选择“上传插件”，然后选择下载的文件。

### 基础指令

| 指令 | 功能描述 |
| :--- | :--- |
| `左对称` | 左半边图像对称到右边 |
| `右对称` | 右半边图像对称到左边 |
| `上对称` | 上半边图像对称到下面 |
| `下对称` | 下半边图像对称到上面 |
| `反色` | 反转图像颜色 |
| `对称帮助` | 显示详细使用说明 |

也可以使用带斜杠的写法，例如 `/左对称`。

**使用示例**：

*   **回复图片**：回复一条含图片的消息，然后发送 `/左对称`。
*   **处理头像**：发送 `/右对称 @某人` 来处理该用户的 QQ 头像。
*   **同消息发送**：在同一消息中发送图片和 `/上对称` 指令。
*   **反色处理**：回复图片或同消息发送图片，然后发送 `/反色`。

### 适配 `qqofficial` / `qqofficial_full` 官方机器人

官方机器人平台无法直接通过 QQ 号取头像，本插件已对此做了适配：

*   **群聊 `GroupMessage`**：使用被 @ 用户的 `member_openid`，头像 URL 为 `https://q.qlogo.cn/qqapp/{AppID}/{member_openid}/640`。若未 @ 他人，则回退使用发送者本人头像。
*   **C2C 私聊 `C2CMessage`**：使用发送者的 `user_openid`，头像 URL 为 `https://q.qlogo.cn/qqapp/{AppID}/{user_openid}/640`。

`AppID` 默认通过 `context.get_platform_inst()` 自动从平台适配器实例读取，无需手动配置；若自动获取失败（如自定义适配器未暴露 `appid` 属性），可在插件配置中手动填写 `qqofficial_appid` 兜底。

## 配置说明

插件提供以下配置选项，可在 AstrBot Web 界面中调整：

| 配置项 | 类型 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| 图像大小限制 (MB) | 整数 | 10 | 普通图像最大大小 |
| GIF 大小限制 (MB) | 整数 | 15 | GIF 动画最大大小 |
| 预检查文件大小限制 (MB) | 整数 | 100 | 打开图像前允许读取的最大文件大小 |
| 压缩最大尺寸 (像素) | 整数 | 2048 | 自动压缩时图像最长边上限 |
| GIF 总像素限制 | 整数 | 16000000 | GIF 帧数乘以单帧像素数的安全上限 |
| 处理超时时间 (秒) | 整数 | 30 | 图像处理超时时间 |
| 输出图像质量 | 整数 | 85 | JPEG/WebP 输出质量 (1-100) |
| 启用 GIF 处理 | 布尔 | true | 是否处理 GIF 动画 |
| 启用自动压缩 | 布尔 | true | 是否自动压缩大图 |
| 静默模式 | 布尔 | true | 是否不发送处理提示 |
| 启用自动清理 | 布尔 | true | 发送后自动清理生成的文件 |
| 文件保留时间 (小时) | 整数 | 1 | 生成文件保留时长，0 表示发送后尽快删除 |
| 启用 @ 用户头像功能 | 布尔 | true | 是否允许通过 @ 用户获取头像处理 |
| QQ官方机器人 AppID (可选) | 字符串 | "" | 仅 qqofficial 系列适配器使用，通常自动读取，无需填写 |
| 请求频率限制 (次/分钟) | 整数 | 10 | 每个用户每分钟最多请求次数，0 表示不限制 |
| 并发处理上限 | 整数 | 3 | 插件同时处理图像任务的最大数量 |
| GIF 最大帧数 | 整数 | 200 | 单个 GIF 允许处理的最大帧数 |
| 清理任务超时时间 (秒) | 浮点数 | 5.0 | 插件卸载时等待清理任务结束的最长时间 |
| 清理循环间隔 (秒) | 整数 | 300 | 后台定时清理生成文件的检查间隔 |

## 项目状态

### 已修复的核心问题

*   **GIF 动画处理**：已修复对称后动画丢失的问题。
*   **版本更新逻辑**：已修复插件市场版本检测异常的问题。
*   **配置系统**：已改为读取 AstrBot WebUI 配置，并补齐 `_conf_schema.json` 中的运行时配置项。
*   **频率限制**：`rate_limit_per_minute = 0` 会按配置说明关闭限制，不再误判为全部请求超限。
*   **高并发保护**：图像处理增加全局并发上限，避免极端并发下无限制创建处理任务。
*   **安全防护**：已实现多层 SSRF、防路径遍历、文件大小、GIF 帧数和总像素限制。

### 安全使用建议

虽然插件已做安全加固，但处理用户提供的网络图片仍存在理论风险。建议在**可信的环境**中使用。对于运行关键服务的服务器，请务必做好网络隔离。
